🔍 Причина: фильтрация relabel_configs применяется ПОСЛЕ обнаружения всех контейнеров

Как работает docker_sd_configs:

1. Promtail подключается к Docker API (tcp://host.docker.internal:2375).
2. Получает список ВСЕХ контейнеров в системе.
3. Для каждого контейнера создаётся target (цель для сбора логов).
4. Только потом применяются relabel_configs → action: keep.

👉 То есть, все контейнеры сначала добавляются, а потом отбрасываются, если не совпадают с regex.

В логе: caller=target.go:133 ... msg="finished transferring logs" written=303 container=52acab76a369...
→ только один контейнер действительно отправил логи.
Остальные — просто обнаружены, но отброшены на этапе relabel_configs.
